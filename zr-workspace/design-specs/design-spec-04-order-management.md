## 2. Screen-by-Screen Analysis

### Screen 112-115: Orders

**Purpose**: View and manage open orders, order history, and trade history

**Layout Structure**:
- **Nav bar**: "Orders" title center, filter icon right
- **Primary tabs** (36px): "Open orders | Order history | Trade history"
- **Filter chips**: "All / Spot / Margin / Futures" horizontal scroll
- **Order list**: Scrollable list of order cards

**Order Card (OKX pattern)**:
```
┌──────────────────────────────────────┐
│ BTC/USDT  Limit  Buy   [Cancel]    │
│ Price: 67,000.00 USDT               │
│ Amount: 0.1500 / 0.1500 BTC        │
│ Filled: 0%          2024-05-07 10:23│
└──────────────────────────────────────┘
```
- Header row: Pair name + Order type badge + Side (green Buy / red Sell) + Cancel button
- Detail rows: Price, Amount (filled/total), Fill percentage, Timestamp
- Expandable to show more details (fee, order ID)

**Interaction Behaviors**:
- Tap "Cancel" -> Confirmation dialog -> Cancel order
- Swipe left on order -> Reveal Cancel button (alternative)
- Tap order card -> Expand to show full details
- Pull to refresh -> Reload orders
- Tap filter chip -> Filter by product type

### Screen 196-199: Trading History

**Purpose**: Historical record of all executed trades

**Layout**:
- Tab within Orders page
- Each trade row: Pair + Side + Price + Amount + Fee + Time
- Filterable by date range and product type

---

## 5. ZR Adaptation

### Direct Adoption
- 3-tab structure: Open / History / Trades
- Order card layout with pair + type + side header
- Swipe-to-cancel interaction
- Filter by product type chips

### Modifications
- Add exchange badge on each order (which exchange executed)
- Add asset class filter (Crypto / Stocks / Forex)
- Add "Export" button for transaction records (SFC compliance)
- Cancel confirmation shows estimated slippage if market has moved
